# 腾讯云开通与部署清单

目标：把三峰整装财务系统的云端 API 部署到腾讯云轻量应用服务器，让桌面端填写云端 API 地址后同步数据。

## 你需要本人完成

1. 注册或登录腾讯云账号。
2. 完成实名认证。
3. 购买轻量应用服务器。
4. 确认付款。
5. 妥善保存服务器密码或 SSH 私钥。

这些步骤涉及账号、实名、付款和登录凭证，必须由账号所有人操作。

## 推荐服务器选择

- 产品：轻量应用服务器 Lighthouse
- 镜像：Node.js 应用模板，或 Ubuntu 后续手动安装 Node.js
- 地域：离使用人员近的地域，例如华东/华南按实际门店位置选择
- 防火墙：
  - `22/TCP`：SSH 登录
  - `8787/TCP`：临时 API 访问
  - `80/TCP` 和 `443/TCP`：后续配置域名和 HTTPS

腾讯云官方文档可参考：

- Node.js 应用模板：https://cloud.tencent.com/document/product/1207/60266
- 轻量服务器 SSH 登录：https://cloud.tencent.com/document/product/1207/44643
- 第三方 SSH 终端登录：https://cloud.tencent.com/document/product/1207/44578
- 管理实例防火墙：https://cloud.tencent.com/document/product/1207/44577
- 管理密钥：https://cloud.tencent.com/document/product/1207/44573

## 拿到服务器后在本机执行

先预检：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\preflight-from-windows.ps1 -HostName 服务器IP -User root
```

如果使用私钥：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\preflight-from-windows.ps1 -HostName 服务器IP -User root -SshKeyPath D:\workspace\your-key.pem
```

预检通过后部署：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\deploy-from-windows.ps1 -HostName 服务器IP -User root
```

私钥方式：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\deploy-from-windows.ps1 -HostName 服务器IP -User root -SshKeyPath D:\workspace\your-key.pem
```

也可以直接使用总控脚本，自动完成预检、部署和公网读写验收：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\deploy-full-from-windows.ps1 -HostName 服务器IP -User root
```

如已准备域名并解析到服务器公网 IP，可一并配置 HTTPS：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\deploy-full-from-windows.ps1 `
  -HostName 服务器IP `
  -User root `
  -Domain api.example.com `
  -Email admin@example.com
```

如果部署或访问异常，收集远程诊断：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\collect-remote-diagnostics.ps1 -HostName 服务器IP -User root
```

## 部署后验证

浏览器打开：

```text
http://服务器IP:8787/api/health
http://服务器IP:8787/api/health/storage
```

或在本机运行完整读写验收：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\verify-remote-api.ps1 -ApiUrl http://服务器IP:8787
```

看到 `ok: true` 后，打开三峰整装财务系统，在登录页填写：

```text
云端 API 地址：http://服务器IP:8787
```

然后注册经销商账户。之后同一经销商代码的数据会保存到云端。

## 来源访问控制

当前桌面客户端使用时，`SANFENG_ALLOWED_ORIGINS` 可以保持为空。若后续增加 Web 管理端，可在服务器编辑：

```bash
sudo nano /etc/sanfeng-cloud-api.env
```

填写允许的浏览器来源：

```env
SANFENG_ALLOWED_ORIGINS=https://api.example.com,https://finance.example.com
```

保存后重启：

```bash
sudo systemctl restart sanfeng-cloud-api
```

## 生产建议

临时可用 `http://服务器IP:8787`。正式使用建议：

1. 绑定域名。
2. 用 Nginx 反向代理到 `127.0.0.1:8787`。
3. 配置 HTTPS。
4. 客户端填写 `https://你的域名`。

如果已经有域名并解析到服务器公网 IP，可在服务器上执行：

```bash
sudo /opt/sanfeng-cloud-api/deploy/setup-nginx-https.sh \
  --domain api.example.com \
  --email admin@example.com
```

执行前确认腾讯云防火墙已放通：

- `80/TCP`
- `443/TCP`

HTTPS 验证：

```bash
curl https://api.example.com/api/health
curl https://api.example.com/api/health/storage
```

HTTPS 完整读写验收：

```powershell
powershell -ExecutionPolicy Bypass -File .\cloud-api\deploy\verify-remote-api.ps1 -ApiUrl https://api.example.com
```

## 自动备份确认

部署脚本会安装每日 03:20 自动备份：

```bash
sudo systemctl status sanfeng-cloud-api-backup.timer --no-pager
sudo systemctl start sanfeng-cloud-api-backup.service
ls -lh /data/sanfeng-finance-backups
```

默认保留最近 14 份备份。恢复时使用：

```bash
sudo SANFENG_CLOUD_DATA_DIR=/data/sanfeng-finance \
  /opt/sanfeng-cloud-api/deploy/restore-data.sh /data/sanfeng-finance-backups/备份文件.tar.gz
```
